Показано всі теми довідника.

TypeScript — це мова на основі JavaScript, яка додає типізацію і допомагає знаходити помилки ще до запуску коду.

Чим відрізняється від JavaScript

JavaScript виконується у браузері напряму, а TypeScript спочатку компілюється у JavaScript. Головна різниця — явні типи для змінних, функцій і об’єктів.

Де використовується

TypeScript часто використовують у фронтенді, веб-додатках, бекенді на Node.js, великих проектах і командах, де важлива стабільність коду.

Перший приклад

let message: string = "Привіт, світ!";
console.log(message);
  • let message створює змінну.
  • : string вказує, що змінна має бути рядком.
  • console.log(message) виводить значення у консоль.

Коментарі

Коментарі такі самі, як у JavaScript: // для одного рядка і /* */ для кількох рядків.

// Однорядковий коментар
/* Багаторядковий
   коментар */

Типізація

Тип пишеться після назви через двокрапку. Це допомагає редактору і компілятору перевіряти код.

let age: number = 18;
let name: string = "Іван";

let і const

let використовують для змінних, які можуть змінюватися. const — для значень, які не переприсвоюються.

let score: number = 0;
score = 10;

const appName: string = "Shop";

Компіляція TS у JS

Браузер виконує JavaScript, тому TypeScript код перетворюють у JavaScript за допомогою компілятора.

tsc app.ts
node app.js

Назва: string

Опис
Рядок тексту.
Приклад
let title: string = "TypeScript";
Коли використовувати
Для імен, повідомлень, описів і будь-якого тексту.

Назва: number

Опис
Числовий тип для цілих і дробових чисел.
Приклад
let price: number = 19.99;
Коли використовувати
Для віку, цін, кількості, розрахунків і координат.

Назва: boolean

Опис
Логічне значення true або false.
Приклад
let isOpen: boolean = true;
Коли використовувати
Для умов, перемикачів і станів.

Назва: any

Опис
Вимикає перевірку типу для значення.
Приклад
let data: any = 10;
Коли використовувати
Тільки тимчасово або для старого коду, бо any прибирає користь TypeScript.

Назва: unknown

Опис
Невідомий тип, який треба перевірити перед використанням.
Приклад
let value: unknown = "text";
Коли використовувати
Для даних ззовні, коли тип ще не відомий.

Назва: void

Опис
Функція нічого не повертає.
Приклад
function log(): void {}
Коли використовувати
Для функцій, які виконують дію без результату.

Назва: null / undefined

Опис
Позначають відсутність значення або ще не задане значення.
Приклад
let user: string | null = null;
Коли використовувати
Коли значення може бути відсутнім і це треба явно показати в типі.

Змінні і union типи

Змінна може мати один тип або кілька дозволених типів через union.

let id: number | string = 10;
id = "user-10";

Функції

У функціях можна типізувати параметри, результат, optional параметри та arrow functions.

function greet(name: string, age?: number): string {
    return `Привіт, ${name}`;
}

const add = (a: number, b: number): number => a + b;

Інтерфейси

interface описує структуру об’єкта: які поля і методи він має мати.

interface User {
    id: number;
    name: string;
    isAdmin?: boolean;
}

const user: User = { id: 1, name: "Оля" };

Типи type

type створює псевдонім типу. Він зручний для union, intersection і складних типів.

type Status = "new" | "done";
type Named = { name: string };
type Worker = Named & { role: string };

Масиви

Масиви типізують через Type[] або Array<T>.

let numbers: number[] = [1, 2, 3];
let names: Array<string> = ["Анна", "Іван"];

Об’єкти і readonly

Об’єкти можна описувати прямо в типі. readonly забороняє зміну поля після створення.

const product: { readonly id: number; title: string } = {
    id: 1,
    title: "Phone"
};

Класи

TypeScript додає модифікатори доступу: public, private, protected і readonly.

class Player {
    constructor(
        public name: string,
        private health: number,
        readonly id: number
    ) {}
}

Generics

Generics дозволяють створювати універсальні функції та типи, які працюють з різними типами без втрати перевірки.

function first<T>(items: T[]): T {
    return items[0];
}

const value = first<number>([10, 20]);

Enum

enum створює набір іменованих значень. Значення можуть бути числовими або строковими.

enum Role {
    User = "USER",
    Admin = "ADMIN"
}

const role: Role = Role.Admin;

Modules

export відкриває значення з файлу, а import підключає його в іншому файлі.

export function sum(a: number, b: number): number {
    return a + b;
}

import { sum } from "./math";

Робота з JavaScript

TypeScript сумісний з JavaScript: JS-код можна поступово переводити на TS, додаючи типи там, де це корисно.

// JavaScript
function add(a, b) {
    return a + b;
}

// TypeScript
function addTyped(a: number, b: number): number {
    return a + b;
}

Типізована функція

function multiply(a: number, b: number): number {
    return a * b;
}

Об’єкт користувача

type User = {
    id: number;
    name: string;
    email?: string;
};

const user: User = { id: 1, name: "Іван" };

Інтерфейс і клас

interface Movable {
    move(): void;
}

class Car implements Movable {
    move(): void {
        console.log("Car moves");
    }
}

Generic функція

function wrap<T>(value: T): { value: T } {
    return { value };
}

const result = wrap<string>("hello");

Enum приклад

enum OrderStatus {
    New = "NEW",
    Paid = "PAID",
    Sent = "SENT"
}

const status: OrderStatus = OrderStatus.Paid;

Назва: interface

Опис
Описує форму об’єкта або класу.
Синтаксис
interface Name { field: Type }
Приклад
interface User { name: string }
Примітка
Інтерфейси можна розширювати через extends.

Назва: type

Опис
Створює псевдонім типу.
Синтаксис
type Name = Type;
Приклад
type Id = number | string;
Примітка
Зручний для union і intersection.

Назва: union

Опис
Дозволяє кілька можливих типів.
Синтаксис
A | B
Приклад
let id: number | string;
Примітка
Перед використанням часто треба перевірити тип.

Назва: intersection

Опис
Об’єднує кілька типів в один.
Синтаксис
A & B
Приклад
type Admin = User & Role;
Примітка
Корисно для комбінування властивостей.

Назва: readonly

Опис
Забороняє змінювати поле після створення.
Синтаксис
readonly field: Type
Приклад
readonly id: number
Примітка
Це перевірка TypeScript на етапі компіляції.

Назва: generic

Опис
Параметр типу для універсального коду.
Синтаксис
function name<T>(value: T): T
Приклад
function id<T>(x: T): T { return x; }
Примітка
Дає гнучкість без втрати типізації.

Назва: enum

Опис
Набір іменованих значень.
Синтаксис
enum Name { A, B }
Приклад
enum Role { User, Admin }
Примітка
У сучасному TS іноді замінюють union рядків.

Назва: public / private / protected

Опис
Модифікатори доступу для класів.
Синтаксис
private field: Type
Приклад
private health: number
Примітка
Допомагають приховувати внутрішні деталі класу.

Назва: export / import

Опис
Працюють з модулями між файлами.
Синтаксис
export ... і import ... from
Приклад
import { sum } from "./math";
Примітка
Синтаксис такий самий, як у сучасному JavaScript.

За вашим запитом нічого не знайдено.
