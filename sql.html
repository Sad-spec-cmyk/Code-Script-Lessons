Показано всі теми довідника.

SQL — це мова для роботи з реляційними базами даних: читання, додавання, зміни, видалення і аналізу даних.

Для чого використовується

SQL потрібен для сайтів, CRM, магазинів, аналітики, звітів, бекенду і будь-яких систем, де дані зберігаються у таблицях.

База даних

База даних — це організоване сховище даних. У реляційних базах дані зазвичай зберігаються у таблицях.

Таблиця

Таблиця схожа на електронну таблицю: має колонки, рядки і значення. Наприклад, таблиця users може мати id, name, age.

Запит

Запит — це команда SQL, яку база даних виконує: знайти користувачів, додати товар, оновити статус або порахувати статистику.

MySQL

Популярна база даних для веб-сайтів і серверних застосунків. Часто використовується разом з PHP, Node.js та іншими технологіями.

PostgreSQL

Потужна реляційна база даних з багатьма можливостями для складних систем, аналітики і надійного бекенду.

SQLite

Легка база даних в одному файлі. Добре підходить для навчання, мобільних додатків і маленьких локальних програм.

SQL Server

Система керування базами даних від Microsoft, часто використовується у корпоративних Windows/.NET середовищах.

Базовий запит SQL читає дані з таблиці і може фільтрувати рядки за умовою.

SELECT name FROM users WHERE age > 18;
  • SELECT name вибирає колонку name.
  • FROM users каже, з якої таблиці брати дані.
  • WHERE age > 18 залишає тільки користувачів старше 18 років.

Вставка даних: INSERT INTO

INSERT INTO додає новий рядок у таблицю.

INSERT INTO users (name, age)
VALUES ('Іван', 25);

Оновлення даних: UPDATE

UPDATE змінює існуючі рядки. Майже завжди потрібен WHERE, щоб не змінити всю таблицю.

UPDATE users
SET age = 26
WHERE name = 'Іван';

Видалення даних: DELETE

DELETE видаляє рядки. Без WHERE може видалити всі дані з таблиці.

DELETE FROM users
WHERE age < 18;

Створення таблиць: CREATE TABLE

CREATE TABLE створює нову таблицю з колонками і типами даних.

  • PRIMARY KEY унікально визначає рядок.
  • AUTO_INCREMENT автоматично збільшує id у MySQL.
  • NOT NULL забороняє порожнє значення.
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    age INT
);

Типи даних SQL

Тип визначає, які значення можна зберігати у колонці.

  • INT — ціле число.
  • VARCHAR — короткий текст змінної довжини.
  • TEXT — довгий текст.
  • DATE — дата.
  • BOOLEAN — true/false.
CREATE TABLE tasks (
    title VARCHAR(150),
    description TEXT,
    done BOOLEAN,
    created_at DATE
);

Фільтрація

Фільтрація залишає тільки потрібні рядки.

  • WHERE задає умову.
  • AND, OR, NOT поєднують умови.
  • LIKE шукає за шаблоном.
  • BETWEEN перевіряє діапазон.
  • IN перевіряє список значень.
SELECT * FROM products
WHERE price BETWEEN 100 AND 500
AND category IN ('phone', 'tablet');

Сортування: ORDER BY

ORDER BY сортує результат. ASC означає за зростанням, DESC — за спаданням.

SELECT * FROM users
ORDER BY age DESC;

Обмеження: LIMIT

LIMIT обмежує кількість рядків у результаті.

SELECT * FROM posts
ORDER BY created_at DESC
LIMIT 10;

Агрегатні функції

Агрегатні функції рахують значення по групі рядків.

  • COUNT рахує кількість.
  • SUM рахує суму.
  • AVG рахує середнє.
  • MIN знаходить мінімум.
  • MAX знаходить максимум.
SELECT COUNT(*) AS users_count,
       AVG(age) AS average_age
FROM users;

GROUP BY

GROUP BY групує рядки і часто використовується з агрегатними функціями.

SELECT category, COUNT(*) AS total
FROM products
GROUP BY category;

JOIN

JOIN поєднує дані з кількох таблиць.

  • INNER JOIN повертає тільки збіги в обох таблицях.
  • LEFT JOIN повертає всі рядки з лівої таблиці.
  • RIGHT JOIN повертає всі рядки з правої таблиці.
SELECT users.name, orders.total
FROM users
INNER JOIN orders ON users.id = orders.user_id;

Індекси

Індекс прискорює пошук і фільтрацію, але може уповільнювати вставку та оновлення.

CREATE INDEX idx_users_email
ON users (email);

Вибір користувачів

SELECT id, name, email
FROM users;

Фільтрація

SELECT *
FROM users
WHERE age >= 18 AND city = 'Kyiv';

Сортування

SELECT name, age
FROM users
ORDER BY age ASC;

JOIN приклад

SELECT users.name, orders.created_at, orders.total
FROM users
LEFT JOIN orders ON users.id = orders.user_id;

Статистика

SELECT status, COUNT(*) AS total_orders, SUM(total) AS revenue
FROM orders
GROUP BY status;

Назва: SELECT

Опис
Вибирає дані з таблиці.
Синтаксис
SELECT columns FROM table;
Приклад
SELECT name FROM users;
Примітка
* означає всі колонки.

Назва: WHERE

Опис
Фільтрує рядки за умовою.
Синтаксис
WHERE condition
Приклад
WHERE age > 18
Примітка
Використовується з SELECT, UPDATE, DELETE.

Назва: INSERT INTO

Опис
Додає новий рядок.
Синтаксис
INSERT INTO table (columns) VALUES (values);
Приклад
INSERT INTO users (name) VALUES ('Іван');
Примітка
Кількість колонок має відповідати кількості значень.

Назва: UPDATE

Опис
Оновлює існуючі рядки.
Синтаксис
UPDATE table SET column = value WHERE condition;
Приклад
UPDATE users SET age = 30 WHERE id = 1;
Примітка
Без WHERE можна змінити всі рядки.

Назва: DELETE

Опис
Видаляє рядки.
Синтаксис
DELETE FROM table WHERE condition;
Приклад
DELETE FROM users WHERE id = 1;
Примітка
Без WHERE можна видалити всі рядки.

Назва: CREATE TABLE

Опис
Створює таблицю.
Синтаксис
CREATE TABLE name (...);
Приклад
CREATE TABLE users (id INT, name VARCHAR(100));
Примітка
Типи даних можуть трохи відрізнятися між базами.

Назва: ORDER BY

Опис
Сортує результат.
Синтаксис
ORDER BY column ASC|DESC
Приклад
ORDER BY created_at DESC
Примітка
ASC часто є значенням за замовчуванням.

Назва: GROUP BY

Опис
Групує рядки для агрегатних обчислень.
Синтаксис
GROUP BY column
Приклад
GROUP BY category
Примітка
Часто використовується з COUNT, SUM, AVG.

Назва: JOIN

Опис
Поєднує таблиці.
Синтаксис
table1 JOIN table2 ON condition
Приклад
users JOIN orders ON users.id = orders.user_id
Примітка
Вибір типу JOIN залежить від потрібних рядків.

Назва: CREATE INDEX

Опис
Створює індекс для швидшого пошуку.
Синтаксис
CREATE INDEX name ON table (column);
Приклад
CREATE INDEX idx_email ON users (email);
Примітка
Індекси варто додавати до колонок, за якими часто шукають.
